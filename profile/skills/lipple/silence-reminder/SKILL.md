# silence-reminder（§3.6）

**機械的・決定論的＝承認なしで実行**（GCP 締切リマインドと同じ配管側）。`scripts/run.py` を cron `--no-agent` で。LLM 非起動。

## ルール
- そのPDCAチャンネルの最後の投稿から **65分** 無音で、次の報告が無ければ、**最終投稿にスレッド返信で1回**。
- 時計の起点＝最後の投稿（種類問わず）。何か来たらリセット。
- **打ち止め**: 当日に「本日の業務を終了します」が出ていればもう鳴らさない（終業後のノート投稿に注意し、最後だけでなく当日全体を見る）。
- **連打しない**: 鳴らした後の追撃なし（`already_reminded_after_ts`）。昼休憩(≒60分)は65分未満で自然にすり抜ける（例外ロジックは持たない）。

## 文面（温かく・理由を添える・@メンション）
「お疲れさまです。最後のご報告から{分}分ほど空いています。1時間ルールの進捗報告、作業中でしたら一言いただけると助かります🙏（手が離せなければ後ほどで大丈夫です）」

## cron
`*/5 * * * *`（65分閾値はスクリプト内。検知 latency ≤5分）。控えは #8902 に流す。

## 既知の調整ポイント（観測知見）
- 松永さんは1時間ルールの進捗報告が薄く、日3〜5回鳴り得る＝本物のシグナル。閾値・静音時間帯・終業欠落時の夜間ハードカットオフは戸田さんの答え合わせで詰める。
